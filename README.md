## 🚫 Hide Taskbar (Prevent accidental popup)

QuickHotkeys can prevent the Windows Taskbar from appearing when you accidentally move your mouse to the bottom edge of the screen.

Normally, Windows shows the Taskbar when you hover the bottom edge — this can interrupt workflow. This feature removes that behavior.

### How it works:

- The Taskbar will NOT open when you move the cursor to the bottom edge of the screen.
- The Taskbar will only open when you hover a **selected screen corner** (default: bottom-right).
- You can still open the Taskbar normally by holding **Ctrl** while hovering the bottom edge.

### Features:

- Prevents accidental Taskbar popups
- Improves full-screen and window management workflow
- Customizable activation corner (bottom-left / bottom-right)
- Can be enabled or disabled with a hotkey

## Hotkey Aliases and Language Switching

How to change language switch hotkeys in Windows? How to assign custom keyboard shortcuts for input languages? This feature allows you to create your own hotkeys for switching between keyboard layouts and input languages, including shortcuts that Windows normally does not allow.

### How to assign custom hotkeys for switching languages?

Windows allows you to assign hotkeys for switching input languages. However, the available key combinations are often inconvenient.

For example, you may want to switch languages using:

* <kbd>Ctrl+F9</kbd> → English
* <kbd>Ctrl+F8</kbd> → Language 2

These combinations are much more convenient than the default Windows shortcuts.

The **Aliases** feature allows one hotkey to emulate another hotkey.

For example:

| Alias Hotkey       | Sends                   |
| ------------------ | ----------------------- |
| <kbd>Ctrl+F9</kbd> | <kbd>Ctrl+Shift+9</kbd> |
| <kbd>Ctrl+F8</kbd> | <kbd>Ctrl+Shift+8</kbd> |

### How to configure language switching

1. Open **Windows Settings** using <kbd>Win+I</kbd>.
2. Search for **Typing**.
3. Open **Typing Settings**.
4. Open **Advanced keyboard settings**.
5. Click **Input language hotkeys**.
6. Select **English** and assign:

   * <kbd>Ctrl+Shift+9</kbd>
7. Select **Language 2** and assign:

   * <kbd>Ctrl+Shift+8</kbd>

After that, return to this program and enable (in Settings):

* **Set English**
* **Set Language 2**

Now pressing:

* <kbd>Ctrl+F9</kbd> will switch to English.
* <kbd>Ctrl+F8</kbd> will switch to Language 2.

### Why use aliases?

Windows does not allow assigning many convenient hotkeys directly to languages. Using aliases makes it possible to use almost any keyboard shortcut for language switching.

### Scope

The **Scope** field limits where the alias works.

For example:

<kbd>CabinetWClass</kbd>

means the alias will only work inside **File Explorer** windows.

## Hold Escape to Shut Down the Computer

### How to Quickly Shut Down a Computer with a Hotkey

Looking for a fast way to shut down your computer?

This feature provides a much faster method.

Simply press and hold the <kbd>Escape</kbd> key for about one second.

A shutdown window will appear with the default action set to **Sleep**.

If Sleep mode is what you need, simply press <kbd>Enter</kbd> and the computer will immediately enter sleep mode.

If you change your mind, press <kbd>Escape</kbd> and the shutdown window will close.

### Benefits

* Quickly open the shutdown dialog.
* Put the computer to sleep in second.
* No need to open the Start Menu.
* Easy to remember.
* Uses a single key.
* Helps reduce repetitive mouse clicks.

### Configuration

The timeout can be adjusted below.

The value is specified in milliseconds.

Examples:

* `500` = 0.5 seconds
* `1000` = 1 second (default)
* `2000` = 2 seconds

The shutdown window will appear only after the Escape key has been held for the specified amount of time.

This helps prevent accidental activation.

### Typical Use Cases

## Sticky Keys

### Press Hotkeys One Key at a Time

**Looking for a better Sticky Keys solution for Windows? Want to press keyboard shortcuts without holding multiple keys at the same time? Need a Sticky Keys alternative that works automatically and releases keys after one second?**

This is one of the most useful features in the entire program.

Once you get used to it, it becomes difficult to work without it.

### What Problem Does It Solve?

Normally, keyboard shortcuts require several keys to be pressed simultaneously.

Examples:

* <kbd>Ctrl+S</kbd> — Save
* <kbd>Ctrl+C</kbd> — Copy
* <kbd>Ctrl+V</kbd> — Paste
* <kbd>Ctrl+Shift+T</kbd> — Restore closed tab
* <kbd>Win+E</kbd> — Open File Explorer
* <kbd>Alt+Tab</kbd> — Switch between applications

The problem is that all of these keys must normally be held down together.

Many users would prefer to press the keys one at a time:

1. Press Ctrl
2. Release Ctrl
3. Press S

Instead of holding both keys simultaneously.

Windows includes a built-in Sticky Keys feature, but many users find it inconvenient.

### Why the Windows Sticky Keys Feature Is Not Ideal

Windows Sticky Keys belongs to the Accessibility features.

To enable it, Windows typically requires pressing <kbd>Shift</kbd> several times.

A beep sounds.

Then a dialog appears asking whether Sticky Keys should be enabled.

Many users encounter this window accidentally and immediately search online for:

* How to disable Sticky Keys
* Stop Sticky Keys popup
* Turn off Sticky Keys notification

The built-in Windows solution is often considered intrusive and inconvenient.

### A Different Approach

This feature works automatically.

There is no need to enable Sticky Keys manually.

There are no popup windows.

There are no warning sounds.

There is nothing to configure before using it.

Sticky Keys are enabled by default.

Simply start pressing hotkeys normally.

### How It Works

When you press a modifier key:

* <kbd>Ctrl</kbd>
* <kbd>Shift</kbd>
* <kbd>Alt</kbd>
* <kbd>Win</kbd>

the key does not immediately release when you release the physical key.

Instead, it remains active for one second.

The default timeout is:

`1000 milliseconds`

You can change this value in the settings.

After the timeout expires, the modifier is automatically released.

### Example: Ctrl+S

Normally:

* Hold Ctrl
* Press S

With Sticky Keys:

* Press Ctrl
* Release Ctrl
* Press S

The program still sends:

<kbd>Ctrl+S</kbd>

and the file is saved.

### Example: Ctrl+Shift+T

Normally:

* Hold Ctrl
* Hold Shift
* Press T

With Sticky Keys:

* Press Ctrl
* Release Ctrl
* Press Shift
* Release Shift
* Press T

The program sends:

<kbd>Ctrl+Shift+T</kbd>

and restores the closed tab.

### One-Handed Hotkeys

This feature makes it possible to use many keyboard shortcuts with one hand.

Instead of stretching your fingers across the keyboard, simply press the keys sequentially.

This is especially useful for:

* Frequent hotkey users

By default:

`1 second`

This is usually enough time to press the next key while preventing accidental activations.

Linux users are often familiar with a feature called:

**Alt Drag**

Simply hold <kbd>Alt</kbd> and drag anywhere inside a window.

Many Linux desktop environments support this natively.

Windows does not.

Because of this, users often install third-party tools such as:

* AltDrag
* AltSnap
* Various AutoHotkey scripts

### A Different Approach

This feature does not use keyboard modifiers at all.

No Alt key.

No Win key.

No Ctrl key.

Only the mouse.

Once learned, it becomes extremely fast.

### Window Moving Mode

To start moving a window:

1. Hold the **right mouse button**.
2. Press and hold the **left mouse button**.
3. Release the **right mouse button**.

Or:

1. Hold the **left mouse button**.
2. Press the **right mouse button**.
3. Release the **right mouse button**.

Now only the left mouse button remains pressed.

Move the mouse.

The window moves.

You can grab the window from virtually any location.

No title bar is required.

### Why This Works Better

Many scripts found online have common problems:

* Text becomes selected accidentally.
* Buttons get clicked accidentally.
* Links activate accidentally.
* Window movement feels jerky.
* Window movement stutters.

This implementation disables interaction with the window while dragging.

As a result:

* No accidental selections.
* No accidental clicks.
* No accidental link activation.
* Smooth dragging.
* Reliable operation.

### Window Resize Mode

To enter resize mode:

1. Press both mouse buttons.
2. Keep the **right mouse button** pressed.
3. Release the **left mouse button**.

Now the window enters resize mode.

Move the mouse to resize the window.

No need to grab the tiny border of the window.

No need to precisely position the cursor on the edge.

### Resize From the Center

A unique feature becomes available while resizing.

If resizing begins from the center area of the window:

* The window is automatically centered on the screen.
* The size changes on both sides simultaneously.
* Left and right edges expand equally.

This makes it easy to create perfectly centered windows.

Useful for:

* Screen recordings.
* YouTube videos.
* Tutorials.
* Demonstrations.
* Professional desktop layouts.

### Full Height Instantly

While in resize mode and holding the right mouse button:

**Mouse Wheel Down**

The window expands to the full height of the screen.

Scroll down again:

The previous size is restored.

### Full Width Instantly

While in resize mode:

**Mouse Wheel Up**

The window expands to the full width of the screen.

Scroll up again:

The previous size is restored.

This is much faster than manually resizing windows.

### Move Windows in a Perfect Straight Line

While moving or resizing:

Hold <kbd>Ctrl</kbd>.

The window becomes constrained to:

* Horizontal movement
* Vertical movement

This makes precise alignment easy.

Especially useful for:

* VSCode layouts.
* Large terminals.
* Side-by-side documents.
* Professional workspace organization.

### Straight-Line Window Movement

Hold <kbd>Ctrl</kbd> while moving a window.

The window will move only horizontally or vertically.

You can release the mouse button, grab the window again at another position, and continue moving it while keeping the same straight-line alignment.

This is useful for moving windows precisely to screen edges, beyond screen boundaries, or for recovering windows that are partially off-screen.


### Recover Windows That Moved Off-Screen

Sometimes a window accidentally ends up:

* Partially off-screen.
* Completely outside the visible desktop.
* On a monitor that is no longer connected.

This is a common Windows problem.

With straight-line movement, you can easily pull windows back onto the screen.

### Window Snapping

While moving or resizing:

Hold <kbd>Shift</kbd>.

The window will snap to:

* Screen edges.
* Other windows.
* The center of the screen.

This makes it easy to create:

* Tiled layouts.
* Symmetrical layouts.
* Perfectly aligned windows.
* Professional desktop arrangements.

### Note

If the cursor is positioned over a web link, the simultaneous left and right mouse button actions are reserved for the **Click on Links** feature.

In that case, the program opens the link in a new tab instead of entering window moving or resizing mode.
